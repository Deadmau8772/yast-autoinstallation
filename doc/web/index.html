AutoYaST2

AutoYaST2 is the system for installing one ore more SuSE Linux systems automatically and without user intervention. Using AutoYaST2, you can create a configuration for a single system or a set of systems to control automated installations. The control file can be provided to YaST2 during installation in different ways.

Major Changes from 8.0 to 8.1

  • Configuration system was adapted to support modular operation. Now the configuration system will automatically allow any additional YaST2 modules which are installed in the system and which have auto-install features.
  • Configuration System offers additional interfaces for configuring the following resources:
    • Sysconfig Variables
    • Complete Configuration files
    • Security Settings
    • Software Selections and additional packages
    • NFS Server/Client configuration
  • RAID Support
  • Better control of installation process
  • Improved Class Management and merge of classes
  • Configuration system now offers the possibility to import the following formats:
    • Alice Configuration files
    • Kickstart files
  • Basic Cloning support
  • Create ISO for a configuration
  • XML parser is now based on libxml2 (unlike Qt XML extensions in 8.0)
  • More partitioning options added. Now it is possible to set filesystem and mount(fstab) options.
  • If confirmation of installation is choosen, the "proposal" interface is shown, where changes can be done to installation values
  • Profile Format Changes - To enable some new features, we had to make some changes to the format of the profile. Here are the major changes:
    • All custom scripts are now under one resource <scripts>
    • Cleaned Partitioning options from unused properties and added comperhensive properties which are easy to identify and understand
    • The general resource has been extended to support more options for the various properties.
